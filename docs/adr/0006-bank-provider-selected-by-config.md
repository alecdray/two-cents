# Bank provider selected by configuration

The bank provider wired in is chosen at the composition root by a single configuration value: the default selects the live provider that reaches the real bank network; one other value selects a deterministic in-process stand-in (fixed accounts and balances, no transactions, no network, no credentials). Everything downstream still depends only on the `BankProvider` seam ([ADR-0002](0002-bankprovider-abstraction.md)) — only which concrete provider is constructed changes.

This **promotes the existing unit-test fake to a wiring option**: when it is selected the real server runs unchanged (real SQLite, HTTP, templ, the real connection flows), and only the leaf provider is swapped. The connection flows span the front end, handlers, persistence, and the provider — a test double substituted inside a unit test exercises none of that integration, and a hosted-provider sandbox needs credentials and returns data too variable for end-to-end assertions. Selecting the fake at the root closes both gaps: the whole stack runs end to end against stable, assertable data with no browser-side mocking and no live bank. Its accounts and balances are deliberately fixed because tests assert on them.

Rejected: exercising the flows only against the live sandbox (needs credentials, makes network calls, non-deterministic data); mocking the provider in the browser (leaves the server's own handling untested); keeping the fake confined to unit tests (never exercises the integration the connection flows actually span).
