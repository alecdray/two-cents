# Income, savings, and full-month drill-ins on the wrap

Extend the month wrap's read-side so the **Income** and **Savings-contributed** figures drill in the same way Category figures already do ([ADR-0009](0009-category-spend-drill-down.md)) — two more `{bucket}` selectors on the one composer-owned drill view, each reconciling to the figure it was reached from — and add an **inline, editable list of the month's whole transaction set** under the wrap's spend-by-Category section.

The drill view's invariant is that the list provably sums to the figure it came from. Two consequences shape this extension:

- **Income reconciles to *gross* income, not "Net income."** The wrap's Net income is total Income − total Spending — a derived net with no single underlying transaction set, so it is not itself a drill. To give income a list that reconciles, the wrap surfaces a distinct **gross Income** figure (the Income legs) as the drill entry point, sitting alongside Net income. Savings contributed already *is* a single set (the savings-contribution source legs), so it reconciles as-is.
- **Income and savings buckets are actuals-only and valid for any month.** Unlike the `everything-else` residual — budget-derived and therefore current-month only — income and savings read no Budget, so their buckets carry no month restriction. The actuals-only reporting projection still owns these figures; the composer only selects and lists the rows behind them.

The **full-month list is not a reconciling bucket** — it spans every classification (income, spending, transfers) and sums to no headline figure — so it is an inline section on the wrap page, not a `{bucket}` on the drill view. Like every transaction list, its rows are edited through the shared transaction-editing modal ([ADR-0011](0011-reusable-transaction-editing-modal.md)): a save announces `transaction-changed` and the affected regions self-refresh ([ADR-0010](0010-event-driven-cross-region-refresh.md)). Because re-categorizing a row there can move money between *any* of the wrap's figures (a Category, gross Income, Savings, and so Net income), the **whole wrap figure region** subscribes to the event — extending the drill's single-region self-refresh to the wrap's full set of figures.

Rejected: drilling the **Net income** figure itself (its list would sum to gross income, not the netted figure clicked — the exact reconciliation break ADR-0009 exists to prevent); a separate income/savings read view (the `{bucket}` selector already serves multi-surface drills — the wrap's income/savings figures here, and the Tracker's Category/everything-else figures); a reconciling "all transactions" bucket (there is no single figure for it to reconcile to).
